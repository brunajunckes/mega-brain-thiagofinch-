## Summary

Implement a multi-source content ingestion engine that supports YouTube channels/videos (with configurable limits), PDFs, plain text/markdown, and images. All content is transcribed, chunked intelligently (500-token chunks with 50-token overlap), converted to embeddings, and stored in per-clone Qdrant collections (`brain_clone_{slug}`).

## Implementation Plan

### Phase 1: Python Chunking Engine
- Create `aiox-engine/brain/ingestion/chunker.py`
  - Token counting via word-based estimation (len(text.split()) * 1.3)
  - 500-token target chunk size
  - 50-token overlap (sentence-aware boundaries)
  - Minimum 50-token chunk enforcement
  - Chunk numbering: `{chunk_index}/{chunk_total}`

### Phase 2: Ingestors (Python)
- Create `aiox-engine/brain/ingestion/youtube.py`
  - yt-dlp for download (supports --playlist-items 1:N for limit)
  - faster-whisper for transcription (base model, local)
  - Chunk pipeline with metadata capture

- Create `aiox-engine/brain/ingestion/pdf.py`
  - PyMuPDF (fitz) text extraction
  - Page tracking in metadata

- Create `aiox-engine/brain/ingestion/doc.py`
  - Plain text and markdown support
  - Heading-aware chunking (prefer breaking at headings)

- Create `aiox-engine/brain/ingestion/image.py`
  - POST to Ollama /api/generate with llava:7b
  - Image description → chunk

### Phase 3: Vector Storage
- Create `aiox-engine/brain/clone/store.py`
  - BrainStore class: parameterized by slug
  - Collection name: `brain_clone_{slug}`
  - Methods: ensure_collection(), upsert_chunk(), search(), get_stats(), delete_collection()
  - Uses existing nomic-embed-text 768-dim embeddings

### Phase 4: REST API
- Create `aiox-engine/brain/api/brain_routes.py`
  - POST `/brain/ingest` → spawn async job, return job_id
  - GET `/brain/job/{job_id}/status` → poll progress
  - GET `/brain/clones` → list all clone collections with stats

- Modify `aiox-engine/api/main.py`
  - Include brain router

### Phase 5: CLI
- Create `bin/aiox-brain.js` (Commander.js entry point)
- Create `bin/modules/brain/index.js` (program definition)
- Create `bin/modules/brain/ingest.js` (ingest subcommand)
- Create `bin/modules/brain/status.js` (status subcommand)
- Create `bin/modules/brain/http-client.js` (shared HTTP client to aiox-engine)

- Modify `bin/aiox.js`
  - Add `case 'brain':` to main switch

### Phase 6: Tests
- Create `aiox-engine/tests/brain/test_chunker.py` (pytest)
- Create `aiox-engine/tests/brain/test_store.py` (pytest, mocked Qdrant)
- Create `tests/brain/brain-cli.test.js` (Jest, mocked HTTP client)

## Technical Notes

- **Chunk sizing:** Token estimation is `len(text.split()) * 1.3` (no tiktoken dependency)
- **Embedding caching:** Redis key `brain:embed:{sha256(chunk[:100])}` prevents re-embedding identical chunks
- **Qdrant collections:** Validated slug format `^[a-z0-9_]{1,40}$` before collection creation
- **YouTube limits:** yt-dlp --playlist-items 1:N where N is `--last` value
- **Whisper model:** Base model (~150MB) cached at `~/.cache/huggingface/hub/`
