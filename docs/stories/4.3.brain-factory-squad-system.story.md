# Story 4.3 — Brain Factory: Squad System

**Epic:** 4 - Brain Factory
**Story ID:** 4.3
**Status:** Ready for Development

## Summary

Implement a Squad System that queries multiple clones in parallel and synthesizes their perspectives into a unified response. Support multi-turn debate between clones, consensus scoring, and perspective summarization. Enables consulting a collective of expert minds on complex questions.

---

## Acceptance Criteria

- [ ] **AC1:** `aiox brain squad --ask "how to scale to 8 figures?" --synthesize` queries all available clones in parallel (30s timeout), returns {clone_name: response} + synthesis
- [ ] **AC2:** Squad queries use same RAG context across all clones (shared vector search, 3 chunks per clone)
- [ ] **AC3:** `--debate 3` flag enables 3-round debate: Round 1 all clones answer, Round 2 each responds to others, Round 3 synthesis
- [ ] **AC4:** Debate responses shorter and more argumentative (prompt-controlled via system instruction change)
- [ ] **AC5:** `--synthesize` flag triggers summary LLM: takes all responses, generates unified perspective + key agreements/disagreements
- [ ] **AC6:** Each clone response includes source chunks in metadata (not visible to user, for synthesis context)
- [ ] **AC7:** `aiox brain squad --list` shows all available clones (name, chunk count, source types)
- [ ] **AC8:** Consensus scoring: identify % of clones agreeing on key points (exported as JSON)
- [ ] **AC9:** Squad results cached at brain:squad:ask:{hash(question)} with 5min TTL
- [ ] **AC10:** Debate history saved to brain:squad:debate:{debate_id} with 24h TTL (can replay)
- [ ] **AC11:** npm run lint passes (zero errors)
- [ ] **AC12:** npm run typecheck passes (zero errors)
- [ ] **AC13:** All unit tests pass (pytest for Python, Jest for Node.js)
- [ ] **AC14:** Performance logged: {num_clones, parallel_queries, synthesis_time, total_tokens}

## Implementation Plan

### Phase 1: Squad Coordinator (Python)
- Create `aiox-engine/brain/squad/coordinator.py`
  - `SquadCoordinator` class: __init__(question, clones=None, timeout=30)
  - `ask_all()`: parallel queries via asyncio.gather(), 30s timeout
  - `debate(rounds=3)`: orchestrate multi-round debate
  - `synthesize()`: call Ollama with all responses → unified answer
  - Methods: get_clones(), set_timeout(), cancel_query()

### Phase 2: Debate System (Python)
- Create `aiox-engine/brain/squad/debate.py`
  - `DebateRound` class: round_num, question, previous_responses, prompt_template
  - `execute_debate_round()`: call each clone with updated prompt
  - `score_consensus()`: analyze agreement across responses (keyword overlap, sentiment)
  - Debate prompt templates: Round 1 (normal), Round 2 (argumentative), Round 3 (synthesis)

### Phase 3: REST API (Python)
- Extend `aiox-engine/brain/api/brain_routes.py`
  - POST `/brain/squad/ask`: accepts {question, clones, use_rag, synthesize, debate_rounds}
  - Returns {all_responses, synthesis, consensus_scores, metrics}
  - GET `/brain/squad/clones`: list all available clones
  - GET `/brain/squad/debate/{debate_id}`: replay debate history

### Phase 4: CLI (Node.js)
- Create `bin/modules/brain/squad.js`
  - Command: `aiox brain squad --ask "question" [--synthesize] [--debate <rounds>]`
  - Flags: --clones, --list, --show-consensus, --json
  - HTTP POST to http://localhost:8000/brain/squad/ask
  - Display options: full responses, synthesis-only, consensus matrix

- Extend `bin/modules/brain/index.js`
  - Add `squad` command to program definition

### Phase 5: Tests
- Create `aiox-engine/tests/brain/test_coordinator.py`
  - test_parallel_queries()
  - test_timeout_handling()
  - test_debate_orchestration()
  - test_synthesis_generation()

- Create `aiox-engine/tests/brain/test_debate.py`
  - test_debate_round_execution()
  - test_consensus_scoring()
  - test_prompt_variation()

- Update `tests/brain/brain-cli.test.js`
  - test_squad_ask_command()
  - test_squad_debate_command()
  - test_squad_list_command()

## Technical Notes

- **Parallel queries:** asyncio.gather() with timeout=30s per clone
- **Debate prompts:** Store in DEBATE_PROMPTS dict with {round: template}
- **Consensus scoring:** TF-IDF keyword extraction + sentiment analysis (basic)
- **Synthesis LLM:** Use qwen2.5:14b for better synthesis (configurable via env)
- **Cache:** brain:squad:ask:{sha256(question)}:all_clones for 5min
- **Debate replay:** Store debate_id → {round, responses, metadata} in Redis
