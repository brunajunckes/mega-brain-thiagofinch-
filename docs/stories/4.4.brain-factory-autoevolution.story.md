# Story 4.4 — Brain Factory: Auto-Evolution

## Summary

Implement Auto-Evolution system that continuously watches YouTube channels for new videos, automatically ingests them into clone knowledge bases, and tracks ingestion history. Supports scheduled polling, incremental updates, and memory reinforcement of frequently cited sources.

---

## Acceptance Criteria

- [x] **AC1:** `aiox brain watch --channel https://youtube.com/@channelname --clone hormozi` sets up watching this channel (stores URL + slug in Redis)
- [x] **AC2:** Watcher daemon polls channels every 6 hours (configurable via env BRAIN_WATCH_INTERVAL), detects new videos automatically
- [x] **AC3:** New videos auto-ingested into clone collection (chunking, embedding, storage) without CLI intervention
- [x] **AC4:** Ingestion history stored at brain:watch:{slug}:history with {video_id, title, chunks_added, timestamp}
- [x] **AC5:** Skip already-seen videos (Redis key brain:watch:seen_videos:{slug} maintains set of ingested video IDs)
- [x] **AC6:** `aiox brain watch --list` shows all monitored channels with last check time and next check time
- [x] **AC7:** `aiox brain watch --clone slug --pause` pauses watching (resumable with --resume)
- [x] **AC8:** Daemon auto-restarts failed ingestions (max 3 retries with exponential backoff)
- [x] **AC9:** Watch events logged to brain:watch:logs:{slug} with {event_type, status, error, duration}
- [x] **AC10:** Graceful shutdown: SIGTERM triggers final sync before exit
- [x] **AC11:** npm run lint passes (zero errors)
- [x] **AC12:** npm run typecheck passes (zero errors)
- [x] **AC13:** All unit tests pass (pytest for Python, Jest for Node.js)
- [x] **AC14:** Performance metrics: {channels_monitored, videos_found, videos_new, ingestion_time, total_tokens}

## Implementation Plan

### Phase 1: Watch Manager (Python)
- Create `aiox-engine/brain/watch/manager.py`
  - `WatchManager` class: manage all active watches
  - `add_watch(channel_url, slug)`: set up new channel watch
  - `list_watches()`: return all active watches
  - `pause_watch(slug)`, `resume_watch(slug)`: control watching
  - Methods: get_watched_channels(), is_watching(slug)

### Phase 2: Watcher Daemon (Python)
- Create `aiox-engine/brain/watch/daemon.py`
  - `WatcherDaemon` class: run in background
  - `start()`: start polling loop
  - `poll_channels()`: check all channels for new videos
  - `check_channel(channel_url)`: use yt-dlp to list latest N videos
  - `ingest_new_videos(slug, video_list)`: run ingestion pipeline
  - Retry logic: exponential backoff (1s, 2s, 4s)
  - Graceful shutdown: catch SIGTERM

### Phase 3: REST API (Python)
- Extend `aiox-engine/brain/api/brain_routes.py`
  - POST `/brain/watch`: add new channel watch
  - GET `/brain/watch`: list all watches
  - PATCH `/brain/watch/{slug}`: pause/resume
  - GET `/brain/watch/{slug}/history`: view ingestion history
  - GET `/brain/watch/{slug}/logs`: view watch events

### Phase 4: CLI (Node.js)
- Create `bin/modules/brain/watch.js`
  - Command: `aiox brain watch --channel <url> --clone <slug>`
  - Flags: --list, --pause, --resume, --show-history, --logs
  - HTTP calls to coordinator daemon

- Create `bin/modules/brain/watch-daemon.js`
  - Standalone process: `aiox brain watch-daemon start|stop|status`
  - Background daemon manager

- Extend `bin/modules/brain/index.js`
  - Add `watch` and `watch-daemon` commands

### Phase 5: Tests
- Create `aiox-engine/tests/brain/test_watch_manager.py`
- Create `aiox-engine/tests/brain/test_daemon.py`
- Update `tests/brain/brain-cli.test.js` with watch tests

## Technical Notes

- **Polling interval:** brain:watch:config:interval (default 6h, env override)
- **Seen videos:** brain:watch:seen_videos:{slug} set (permanent, no TTL)
- **Watch state:** brain:watch:{slug}:state {active: true/false, paused: timestamp}
- **History:** brain:watch:{slug}:history list (newest first, no TTL)
- **Daemon PID:** brain:watch:daemon:pid (used for status checks)
- **Graceful shutdown:** SIGTERM → final sync → exit (no data loss)
