## Summary

Implement a Clone Engine that loads expert personas from the Brain Factory (outputs/minds/{slug}/implementation/system-prompt.md), builds RAG-enabled clone agents that query per-clone Qdrant collections, and provides a CLI interface to consult individual clones. Each clone maintains conversation context via Redis session storage and answers in-persona.

## Implementation Plan

### Phase 1: Clone Persona Builder (Python)
- Create `aiox-engine/brain/clone/builder.py`
  - `load_persona(slug)`: reads outputs/minds/{slug}/implementation/system-prompt.md
  - `get_system_prompt(slug)`: returns cached persona from Redis or filesystem
  - Graceful fallback for missing persona
  - Cache in Redis with 1h TTL

### Phase 2: Clone Agent (Python)
- Create `aiox-engine/brain/clone/agent.py`
  - `CloneAgent` class: __init__(slug, session_id=None)
  - `ask(question, use_rag=True)`: orchestrate RAG + Ollama call
  - RAG pipeline: query BrainStore → format context → build messages array
  - Call Ollama /api/chat with system_prompt + context + question
  - Cache responses in Redis (30min)
  - Track metrics: tokens, chunks, latency

### Phase 3: Session Management (Python)
- Extend `aiox-engine/session/manager.py` (already exists from Story 4.1)
  - Add clone-specific session key: `brain:session:{session_id}:{slug}`
  - Store conversation turns: [{role, content, timestamp}, ...]
  - 24h TTL per session
  - Methods: save_clone_message(), get_clone_history(), clear_clone_session()

### Phase 4: REST API (Python)
- Create/extend `aiox-engine/brain/api/brain_routes.py`
  - POST `/brain/ask`: accepts {slug, question, session_id, use_rag}
  - Returns {slug, response, session_id, chunks_used, cache_hit, timestamp}
  - GET `/brain/ask/{slug}/history`: returns last N messages from session

### Phase 5: CLI (Node.js)
- Create `bin/modules/brain/ask.js`
  - Command: `aiox brain ask --clone {slug} {question}`
  - Flags: --session, --history, --stream, --model
  - HTTP POST to http://localhost:8000/brain/ask
  - Displays response with attribution metadata

- Extend `bin/modules/brain/index.js`
  - Add `ask` command to program definition

### Phase 6: Tests

  - test_load_persona_from_filesystem()

## Architecture

```
CLI (Node.js)                   Python Backend (FastAPI)
bin/modules/brain/ask.js        aiox-engine/brain/clone/
  ask --clone <slug>              ├─ builder.py (load_persona, get_system_prompt)
         ↓ HTTP POST              ├─ agent.py (CloneAgent.ask, session management)
    /brain/ask          →          ├─ store.py (BrainStore, per-clone Qdrant)
                                   └─ api/ (brain_routes.py)
```

**Flow:**
1. User runs: `aiox brain ask --clone alex_hormozi "question"`
2. CLI POSTs to `/brain/ask` with {slug, question, session_id, use_rag}
3. Backend creates CloneAgent(slug)
4. Agent loads persona from Redis or outputs/minds/{slug}/
5. Agent queries BrainStore.search(question) → top-3 chunks
6. Agent builds: system_prompt + context + question
7. Agent calls Ollama /api/chat
8. Response cached in Redis (30min)
9. Session saved to Redis (24h)
10. Response returned to CLI with metadata

## Technical Notes

- **Persona loading:** Search outputs/minds/{slug}/implementation/system-prompt.md, fallback to generic prompt
- **RAG context:** Top-3 chunks from BrainStore.search(), formatted as Context: {chunk1}\n{chunk2}\n{chunk3}
- **Session isolation:** Redis key includes both session_id AND slug to prevent cross-clone contamination
- **Streaming:** Ollama /api/chat stream=true if client requests (ACx mentions streaming option)
- **Metrics:** Log {slug, input_tokens, output_tokens, chunks_used, latency_ms} to Redis for future analysis
