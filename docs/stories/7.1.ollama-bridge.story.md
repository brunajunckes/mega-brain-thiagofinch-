## 📋 Overview

Implementar **AIOX Ollama Bridge** — wrapper CLI que roteia automaticamente tarefas entre Ollama (local, gratuito) e Claude API (pago) baseado na complexidade detectada. Permite usar Ollama para tarefas simples (leitura, busca, documentação) economizando tokens do Claude para trabalho mais complexo.

**O que faz:**
- Classifica complexidade de mensagens por heurísticas (sem LLM)
- Roteia tarefas simples (score 1-3) para Ollama
- Roteia tarefas complexas (score 4-5) para Claude
- Interface interativa com comandos especiais (`/modelo`, `/threshold`, `/force-ollama`)
- Integra com contexto AIOX (stories ativas, regras, branch atual)
- Acessível via `claude-ollama` ou `aiox ollama`

**Token Estimate:** 8-10k
**Tempo Estimado:** 4-5h

## 🏗️ Implementation Plan

### Phase 1: Cliente Ollama
- `packages/ollama-bridge/src/client.js` — HTTP client puro (sem deps)
- Endpoints: `/api/chat`, `/api/tags`, `/api/pull`
- Health check e streaming

### Phase 2: Classificador de Complexidade
- `packages/ollama-bridge/src/classifier.js` — heurísticas
- Score 1-5 baseado em: palavras-chave, length, files mencionadas, story mentions

### Phase 3: Config & Context
- `packages/ollama-bridge/src/config.js` — carregar/salvar config
- `packages/ollama-bridge/src/context.js` — carregar stories, rules, branch

### Phase 4: Router
- `packages/ollama-bridge/src/router.js` — decide Ollama vs Claude

### Phase 5: Chat Interativo
- `packages/ollama-bridge/src/chat.js` — readline loop com streaming
- Comandos especiais

### Phase 6: Entry Points
- `bin/aiox-ollama.js` — CLI standalone
- Modificar `bin/aiox.js` para suportar `ollama` subcommand
- `/usr/local/bin/claude-ollama` — script global
