Brand Motion System

Motion

Design em movimento

O movimento da Hubme.Ai é preciso, intencional e nunca supérfluo. Cada animação comunica estado, não decoração.

Purposeful
Toda animação tem razão de existir
Precise
Valores exatos, não aproximações
Calm
Nunca ansioso, nunca excessivo
Informative
Mudança de estado sempre animada
01 //

Princípios

Quatro regras que governam todo movimento na interface. Violá-las significa adicionar ruído, não clareza.

01 — PRINCÍPIO
Intencional
Anime para comunicar, nunca para impressionar. Cada motion precisa responder: "que informação isso transmite?"
Hover revela contexto — feedback preciso, sem exagero
02 — PRINCÍPIO
Preciso
Use tokens exatos. "ease-in-out" arbitrário é diferente de var(--ease-neutral).
ease-out (correto)
linear (robotico)
Mesma distância, easing diferente — percepção completamente distinta
03 — PRINCÍPIO
Efêmero
Movimentos devem completar rapidamente. O usuário não veio ver animações — veio usar o produto.
150ms
800ms
150ms vs 800ms — o primeiro serve, o segundo irrita
04 — PRINCÍPIO
Informativo
Mudanças de estado precisam ser percebidas. Um toggle sem animação gera dúvida: "funcionou?"
off
Clique o toggle — a animação confirma a ação
02 //

Curvas de Easing

Três curvas, cada uma com propósito específico. Escolha pela intenção — não pela preferência pessoal.

Default
ease-out
Entrada rápida, saída suave
cubic-bezier(0.16, 1, 0.3, 1)
Para elementos entrando na tela. Parece natural porque imita física real — atrito no final.
Spring
spring
Pequeno bounce. Alta energia.
cubic-bezier(0.34, 1.56, 0.64, 1)
Confirmações, toasts, botões de ação principal. Comunica que algo aconteceu com sucesso.
Neutral
ease-in-out
Transição simétrica e equilibrada
cubic-bezier(0.4, 0, 0.2, 1)
Modais, sidebars, transitions de página. Entrada e saída com igual peso.
03 //

Durações

A escala de tempo da Hubme.Ai. Use tokens CSS, nunca valores arbitrários. Hover em cada linha para ver a animação ao vivo.

--dur-instant 80ms Tooltips, cursor changes, micro-feedback
--dur-fast 150ms Buttons, inputs, small state changes
--dur-normal 250ms Cards, navigation, content reveals
--dur-slow 400ms Page transitions, modals, drawers
--dur-enter 600ms Hero sections, landing sequences, choreography
motion-tokens.css 
:root {
  /* Duration Scale */
  --dur-instant:  80ms;   /* tooltips, cursor */
  --dur-fast:     150ms;  /* buttons, inputs */
  --dur-normal:   250ms;  /* cards, nav */
  --dur-slow:     400ms;  /* modals, pages */
  --dur-enter:    600ms;  /* hero, sequences */

  /* Easing Curves */
  --ease-out:     cubic-bezier(0.16, 1, 0.3, 1);
  --ease-spring:  cubic-bezier(0.34, 1.56, 0.64, 1);
  --ease-neutral: cubic-bezier(0.4, 0, 0.2, 1);
}
04 //

Keyframes

As cinco animações nativas da Hubme.Ai. Cada uma tem um propósito semântico específico — não são decorativas, são funcionais.

hub-pulse
Logotipo respirando. Presença viva, não mecânica.
@keyframes hub-pulse {
  0%, 100% { transform: scale(1); opacity: 1; }
  50%      { transform: scale(1.08); opacity: 0.82; }
}
/* usage */
animation: hub-pulse 2.5s var(--ease-neutral) infinite;
hub-ping
Radar ping. Sinal de atividade ou notificação.
@keyframes hub-ping {
  0%        { transform: scale(1); opacity: 0.7; }
  75%, 100% { transform: scale(2.4); opacity: 0; }
}
/* usage */
animation: hub-ping 1.8s cubic-bezier(0,0,0.2,1) infinite;
hub-enter
Elemento entrando de baixo com opacity. A animação padrão de aparecimento.
Elemento A
Elemento B
Elemento C
@keyframes hub-enter {
  from { opacity: 0; transform: translateY(12px); }
  to   { opacity: 1; transform: translateY(0); }
}
/* usage */
animation: hub-enter 600ms var(--ease-out) both;
hub-spin-slow
Rotação lenta e contínua. Para loading states e indicadores de processo.
Processando...
@keyframes hub-spin-slow {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}
/* usage */
animation: hub-spin-slow 2s linear infinite;
hub-glow-pulse
Glow pulsando em elementos de accent. Comunica "ativo" ou "disponível".
@keyframes hub-glow-pulse {
  0%, 100% {
    box-shadow:
      0 0 8px 0 rgba(34,211,238,0.15),
      0 0 24px 0 rgba(34,211,238,0.05);
  }
  50% {
    box-shadow:
      0 0 20px 0 rgba(34,211,238,0.4),
      0 0 60px 0 rgba(34,211,238,0.15);
  }
}
/* usage */
animation:
  hub-glow-pulse 2.5s var(--ease-neutral) infinite;
05 //

Coreografia

Como sequenciar múltiplos elementos entrando na tela. Stagger cria percepção de fluidez sem parecer aleatório.

→
Stagger delay: cada item adiciona +80ms de delay ao anterior
≤
Máximo 5 elementos simultâneos. Acima disso, quebre em grupos.
↓
Direção: sempre de baixo para cima (translateY positivo → zero). Nunca lateral sem motivo.
t
Use animation-fill-mode: both para evitar flash antes/depois da animação. 
/* Stagger pattern */
.item:nth-child(1) { animation-delay: 0ms; }
.item:nth-child(2) { animation-delay: 80ms; }
.item:nth-child(3) { animation-delay: 160ms; }
.item:nth-child(4) { animation-delay: 240ms; }
.item:nth-child(5) { animation-delay: 320ms; }

.item {
  animation: hub-enter var(--dur-enter) var(--ease-out) both;
}
Conexão estabelecida 0ms
Dados sincronizados +80ms
Agentes inicializados +160ms
Pipeline ativo +240ms
Sistema pronto +320ms
06 //

Do & Don't

Os erros mais comuns. Lado a lado para deixar a diferença inegável.

DO
Feedback imediato — 150ms
O usuário sente que a UI responde a ele
Hover mostra resposta em ~150ms. Parece vivo.
DON'T
Feedback lento — 800ms
O usuário duvida se clicou
Hover demora 800ms. Parece travado ou com bug.
DO
ease-out para entradas
Física natural — rápido no começo, suave no fim
Desacelera organicamente. Parece intencional.
DON'T
linear — velocidade constante
Parece mecânico, programático, inumano
Velocidade constante. Parece um script rodando.
DO
Movimento sutil e preciso
Hover com translateY(-3px) + scale(1.02)
Hover sobre o bloco
Leve elevação confirma interatividade. Não distrai.
DON'T
Bounce excessivo e contínuo
Animação infinita sem parar = ansiedade visual
Isso nunca para. Irritante.
Rotação + scale + loop infinito. Grita por atenção o tempo todo.
